# 2차 빌드 프롬프트

> 이 프롬프트는 1차 빌드로 만든 노트들을 *일관성 카드 5규칙*에 맞춰 재정렬합니다.
> 내용은 한 글자도 안 바꿉니다. 구조·메타데이터·연결만 손봅니다.

---

지금 이 vault에 1차 빌드로 만든 노트들을 일관성 카드 5규칙에 맞춰 2차로 재구성해줘.
내용은 건드리지 말고 *구조·메타데이터·연결*만 손봐.

== 사용자 페르소나 ==

페르소나: **{persona}**
- who 라벨: {who_label}
- topic 라벨: {topic_label}
- kind 예시: {kind_examples}

== 일관성 카드 5규칙 ==

1. **네이밍** — 노트 제목에 검색어를 포함시킨다.
   원칙: 추상 제목 → 검색어·고유번호·날짜가 박힌 구체 제목.
   본인 vault 어휘에 맞춰 변환 ({who_label}·식별자·날짜 중
   *식별자가 되는 것*을 제목에 넣음).

2. **태그** — 4축으로 properties에 박는다.
   - `time`: <시점>
   - `who`: <주체 — {who_label}>
   - `topic`: <{topic_label} 1~2단어>
   - `kind`: <{kind_examples} 중 하나>

3. **링크 방향** — *구체 → 추상* 한 방향만 허용.
   사례 노트가 기준·근거·참고 자료 노트를 [[wikilink]]로 가리킨다.
   기준·근거 노트는 사례를 백링크로 자동 표시받기만 한다.

4. **요약 1줄** — 모든 노트의 첫 줄을 다음 형식으로 강제.
   "이 노트는 X에 대한 Y이다."
   X = 대상, Y = 문서 종류

5. **불완전 마커** — 검토 필요·미확정 부분에는 본문에 `[확인 필요]`
   또는 `[질문]` 인라인 표시. 모호한 표현 자동 삭제 금지 —
   본인이 직접 판단할 자리이므로 마커로만 표시.

== 작업 절차 ==

1) vault 안 모든 `.md` 노트를 스캔하고 위 5규칙 위반 항목을
   `_meta/2차빌드_점검.md`에 표로 먼저 보고해줘.
   컬럼: `노트경로 | 위반규칙 | 현재값 | 제안값`

   옵시디언 CLI 활용:
   ```
   obsidian files folder=. ext=md
   obsidian property:read name=time path=<file>
   ```

2) 내가 표를 본 뒤 "진행"이라고 답하면 그때 각 노트에 변경을 적용.
   일괄 적용 전에 멈춰.

3) 적용 시 옵시디언 CLI 활용:
   ```
   obsidian rename path=<old> name=<new>
   obsidian property:set name=<key> value=<val> path=<file>
   obsidian append path=<file> content=<first-line-summary>
   ```

4) 적용 후 그래프뷰 변화가 어디에서 일어났는지 한 줄로 요약.
   (어떤 노드가 새 링크를 받았는지, 고립 노드가 줄었는지)

5) 적용 후 옵시디언 자동 새로고침:
   ```
   obsidian reload
   obsidian command id=graph:open
   ```

== 보호 규칙 ==

- 본문 내용·숫자·고유명사는 한 글자도 바꾸지 마.
- `raw/` 폴더 원본 자료는 건드리지 마.
- properties만 추가·수정. 본문 문장은 첫 줄 요약 추가 외에는 유지.
- 불확실하면 `[확인 필요]` 마커만 박고 추측 금지.
- 사용자 "진행" 답 받기 전 일괄 적용 금지.
