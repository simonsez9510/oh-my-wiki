# `_after_example/` — 2차 빌드 후 결과 미리보기

이 폴더는 **`raw/`의 3개 자료가 1차·2차 빌드를 거치면 어떤 모양으로 떨어지는지** 보여주는 *참고용* 결과물입니다.

## 왜 별도 폴더인가

스킬은 `클라이언트/`, `회계기준/`을 vault 루트에 새로 만듭니다. 이 폴더(`_after_example/`)는 그 경로와 겹치지 않으므로 `/oh-my-wiki:bootstrap` 실행해도 덮어쓰이지 않습니다. 데모 vault에 그대로 두고 비교 자료로 쓰세요.

## 안에 들어있는 것

```
_after_example/
├── 클라이언트/
│   ├── (주)가나_2025감사메모.md       (raw/01 → 사례 노트)
│   └── (주)다라_2026-05-20미팅.md     (raw/03 → 사례 노트)
└── 회계기준/
    └── 수익인식 (K-IFRS 1115).md      (raw/02 → 기준 노트)
```

3개 노트 모두 일관성 카드 5규칙(네이밍·태그 4축·구체→추상 링크·1줄 요약·불완전 마커) 적용 후 모양입니다.

## 그래프뷰에서 기대되는 모양

- `(주)가나_2025감사메모` → `수익인식 (K-IFRS 1115)` 위키링크
- `(주)가나_2025감사메모` → `리스 (K-IFRS 1116)` **고스트 노트** (raw 자료 없음)
- `(주)다라_2026-05-20미팅` → `연결범위 (K-IFRS 1110)` **고스트 노트**
- `수익인식 (K-IFRS 1115)` 는 `(주)가나_2025감사메모` 로부터 백링크를 자동 표시받음 (옵시디언 사이드 패널)

*사례 → 기준* 한 방향 화살표 클러스터.
