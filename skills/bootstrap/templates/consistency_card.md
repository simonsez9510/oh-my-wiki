# 일관성 카드 — 5규칙

> 이 카드는 vault 안 모든 노트가 따라야 하는 *최소 규칙*입니다.
> LLM이 매번 이 규칙을 외워 보낼 필요 없이, 이 카드 한 장이 vault 안에 있으면 됩니다.

## 5규칙 (원형)

1. **네이밍** — 노트 제목에 검색어 포함
2. **태그** — 시점·주체·주제·문서종류 4축
3. **링크 방향** — 구체 → 추상 (사례가 기준 가리킴)
4. **요약 1줄** — 첫 줄에 "이 노트는 X에 대한 Y이다"
5. **불완전 마커** — `[확인 필요]`, `[질문]`

## 규칙 풀이

| 규칙 | 풀이 |
|---|---|
| **1. 네이밍** | 추상 제목 → 검색 가능한 식별자가 박힌 구체 제목. 본인 도메인 어휘(클라이언트명·의뢰인명·취재원명·프로젝트명)로 변환. |
| **2. 태그 4축** | `time`(시점) · `who`(주체) · `topic`(주제) · `kind`(문서종류). properties에 박으면 obsidian-bases에서 표로 필터링 가능. |
| **3. 링크 방향** | *구체 → 추상* 한 방향. 사례 노트가 기준·근거 노트를 가리키고, 기준은 사례를 안 가리킴. 옵시디언이 백링크 패널로 *역방향*을 자동 보여줌. |
| **4. 요약 1줄** | "이 노트는 X에 대한 Y이다." X=대상, Y=문서 종류. 모든 노트 첫 줄에 강제. 나중에 검색·요약할 때 결정적. |
| **5. 불완전 마커** | `[확인 필요]`·`[질문]` 인라인. 모호한 표현 자동 삭제 금지 — 본인이 판단할 자리이므로 마커로만. |

## 보호 규칙 (LLM이 따라야 함)

- 본문 내용·숫자·고유명사는 한 글자도 바꾸지 마.
- `raw/` 폴더 원본 자료는 건드리지 마.
- properties만 추가·수정. 본문 문장은 첫 줄 요약 추가 외에는 유지.
- 불확실하면 `[확인 필요]` 마커만 박고 추측 금지.
